Generator pages in constexpr.js

  • constexpr.js
  • web

Generator pages is a feature in constexpr.js that allows you to generate an arbitrary number of HTML pages. For example, you might want to generate a webpage for each tag in your blog, or you might want to paginate the list of posts in your blog.

The main entry point for this feature is the window._ConstexprJS_.addPaths function. By default, constexpr.js compiler creates one output page for each html file in the input directory (unless abort is called). addPaths takes an array as input and adds an additional page for each entry in that array. The array elements must be objects with two keys, generator and output:

window._ConstexprJS_.addPaths( [ { generator: "/posts/generator_from_date.html?2021-jan", output: "/posts/2021-jan.html" }, { generator: "/posts/generator_from_date.html?2021-feb", output: "/posts/2021-feb.html" } ] )

This tells the compiler to render the page /posts/generator_from_date.html?2021-jan and save it's result to /posts/2021-jan.html.

Think of the generator page as a function that takes input in url parameters/hash and produces the intended page as output. Try to make this a pure function.

Generator pages are used to build the tag specific pages on this website.

Links to generated pages

The compiler also maps all the links to generator page (/posts/generator_from_date.html?2021-jan) to the correct output HTMLs (/posts/2021-jan.html). This ensures that the original and the final website look and work exactly the same when you use this feature.
